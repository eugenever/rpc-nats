# RPC implementation based on the NATS core

Вариант реализации RPC на основе `NATS core` посредством концепции [Request-Reply](https://docs.nats.io/nats-concepts/core-nats/reqreply).
При необходимости `NATS core` можно использовать как `embedded` решение в рамках собственных микросервисов, написанных на Go.

Данный подход позволяет реализовать архитектуру из нескольких серверов `NATS` (обеспечение отказоустройчивости) и целевых процессов-воркеров на разных языках программирования, имеющих клиентские библиотеки для `NATS`.
В указанном варианте RPC необязательно использовать `protobuf` для обмена сообщениями.
